<h2 align=center>3 common Enterprise Root Causes of Time-based One-Time Password (TOTP) failures</h2>

| Root cause                                     | Why it causes failure                                                                                                                                                     | Diagnostic step                                                                                                                                                                                                   |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Clock synchronization drift**                | TOTP codes are generated from the current time. If the user's device or the authentication server has significant time drift, the generated code won't match.             | Compare the device and server time against a reliable NTP source and verify that both are synchronized                               |
| **Incorrect or mismatched shared secret**      | The authenticator app and the authentication server must use the same secret key. A failed enrollment, re-provisioning, or account reset can result in different secrets. | Re-enroll the user's TOTP token (or compare the stored secret with the enrollment record, if operational procedures permit) and verify that newly generated codes authenticate successfully.                      |
| **User entering an expired or incorrect code** | TOTP codes are valid only for a short period (commonly 30 seconds). Delays, mistyped digits, or confusion between multiple accounts can result in an invalid code.        | Review authentication logs for repeated invalid TOTP attempts and ask the user to generate a fresh code immediately before entering it, confirming they are using the correct account in their authenticator app. |
