<h2 align="center">Mapping WireGuard Components to InstaSafe ZTNA Components</h2>

| WireGuard Component                   | Purpose                                                                   | InstaSafe ZTNA Component                                    | Purpose                                                                                          |
| ------------------------------------- | ------------------------------------------------------------------------- | ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| **WireGuard Client**                  | Device endpoint that initiates the encrypted tunnel                       | **InstaSafe Agent**                                         | Installed on the user device; initiates secure access requests and connects to the ZTNA platform |
| **WireGuard Server**                  | Terminates WireGuard tunnels and forwards traffic                         | **InstaSafe Gateway**                                       | Provides secure access to protected applications/resources and handles traffic forwarding        |
| **WireGuard Tunnel**          | Encrypted point-to-point path between WireGuard peers                     | **ZTNA Secure Connection / Session**                        | Encrypted communication path established between Agent and Gateway after authorization           |
| **WireGuard Public/Private Keys**     | Cryptographic identity used for peer authentication and tunnel encryption | **Certificates / Device Identity Credentials**              | Used to verify user/device identity, establish trust, and enable secure communication            |
| **WireGuard Endpoint**   | Destination address where the client connects                             | **InstaSafe Gateway Endpoint**                              | Network endpoint where the Agent establishes the secure connection                               |
| **Handshake**                         | Cryptographic session establishment between WireGuard peers               | **Controller-assisted authentication and connection setup** | Controller validates identity, applies policy, and coordinates an authorized connection          |
| **Peer Configuration (`AllowedIPs`)** | Defines what networks a peer can access                                   | **ZTNA Access Policies**                                    | Defines which applications/resources a user is allowed to access                                 |
| **Firewall rules / routing**          | Controls traffic flow after tunnel creation                               | **Gateway enforcement and segmentation policies**           | Enforces approved access paths and restricts unauthorized access                                 |
